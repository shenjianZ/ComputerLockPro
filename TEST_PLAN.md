# ComputerLock Pro 完整测试文档

## 1. 测试目标

本文档用于完整验证 ComputerLock Pro 当前代码中已经实现的所有功能，覆盖主窗口、路由、锁屏、解锁、系统托盘、全局快捷键、密码管理、系统设置、专注模式、安全防护、电源管理、安全日志、本地数据持久化和关键异常路径。

测试结论必须以实际结果为准。任何失败项都需要记录：测试编号、环境、复现步骤、期望结果、实际结果、日志或截图。

## 2. 测试环境

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11 |
| 运行方式 | `pnpm tauri dev` 或安装包 |
| 前端地址 | `http://localhost:1420/` |
| 数据库 | `~/.computer-lock-pro/computer-lock.db` |
| 权限 | 测试 hosts、开机自启、关机计划时建议使用管理员权限 |
| 多显示器 | 至少准备单屏；多屏功能需准备 2 个及以上显示器 |

## 3. 测试前准备

1. 确认依赖安装完成：`pnpm install`。
2. 启动应用：`pnpm tauri dev`。
3. 准备一个普通测试文件作为 USB Key，例如 `D:\ComputerLockTest\usb.key`。
4. 准备一张本地图片作为壁纸锁屏测试图片，例如 PNG/JPG/WEBP。
5. 如需测试网站限制，先备份 `C:\Windows\System32\drivers\etc\hosts`。
6. 如需测试倒计时关机，使用较短时间并准备执行取消计划。

## 4. 全局验收标准

| 编号 | 验收点 | 通过标准 |
| --- | --- | --- |
| G-01 | 应用启动 | 主窗口正常打开，无启动崩溃，数据库自动创建或复用 |
| G-02 | UI 响应 | 页面切换无明显卡顿，按钮状态在数据返回后正确 |

| G-05 | 错误处理 | 无效输入不会导致崩溃，界面能展示错误提示 |
| G-06 | 控制台 | 无持续刷屏的前端异常或 Tauri permission denied |

## 19. 已知平台限制

1. Ctrl+Alt+Del、UAC 安全桌面、任务管理器的高权限终止进程、远程管理等属于 Windows 系统级能力，普通用户态应用无法做到登录界面级别的强制锁定。
2. hosts 修改、开机自启、关机计划、输入锁定和部分蓝牙查询依赖 Windows 权限与系统策略，失败时需要记录错误信息和权限状态。
3. 休眠计划由后台线程延迟触发，应用退出后该线程不会继续执行；关机计划由系统 `shutdown` 维护，可通过取消计划撤销。
4. 当前主题选择已持久化，但界面是否即时完全应用 light/dark/system 需要按实际实现记录。
